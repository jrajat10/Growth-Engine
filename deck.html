03 / 07
Current Capabilities

What the project can do now

📊
Constrained Budget Optimizer
Scores channels by net-new efficiency × LTV multiplier. Enforces floor, cap, CAC ceiling, and max 15% weekly shift. Finance sets the envelope — math decides where it goes.
📋
Auto-Generated Weekly Brief
Headline KPIs, top 3 priority actions ranked by impact, counterfactual ("approving last week's rec would have cut CAC 4–7%"), and pending decisions queue.
💬
Natural Language Q&A
20+ query patterns — "what's the worst channel?", "should we stay in LATAM?", "what experiment is ready?" Answered from live state data. Zero API cost, zero hallucination.
🗺
Geo-Aware Scoring
ARPU multipliers per region (LATAM 0.45×, US West 1.15×). A cheap LATAM CAC still loses to a pricier US West CAC after LTV adjustment — the optimizer knows this.
🧪
Experiment Tracking
Tracks A/B tests and geo holdouts. Surfaces p-value, lift %, and recommendation (ship / stop / keep running) directly in the weekly brief.
📁
Immutable Decision Ledger
Every approval written as append-only JSONL — never overwritten. Captures segment, week, recommended allocation, human action, and timestamp. Compounding institutional memory.
04 / 07
Honest Assessment

What the project cannot do yet

No live data connection
Runs on CSV sample fixtures. Not wired to Google Ads API, Meta API, LinkedIn, or any data warehouse (Snowflake / BigQuery).
No authentication or access control
Single-user, no RBAC. Anyone with the URL can approve decisions.
NL query is pattern-matching, not conversational
20 hardcoded patterns. Questions outside those patterns get a generic fallback — not true natural language understanding.
Optimizer uses heuristic scoring, not ML
Efficiency × LTV × CAC penalty — transparent and explainable, but not a trained model. No predictive power on future channels.
Attribution is last-touch only
Markov chain attribution is scaffolded but not implemented. Multi-touch credit is not yet distributed correctly.
No trend forecasting
Trends API endpoint exists, but no ML model underneath. Historical data shown — forward projections not generated.

Future Vision

Roadmap — 4 phases

Phase 1 0 – 3 months
Live Data Wires
  • Connect Google Ads, Meta & LinkedIn APIs
  • Replace CSV fixtures with Snowflake / BigQuery reads
  • Daily data refresh, not weekly snapshots
  • OAuth login + role-based access control
  • Slack / email weekly brief delivery
Phase 2 3 – 6 months
ML Scoring Layer
  • Train a proper channel efficiency model (Bayesian MMM or gradient boosted)
  • Feed ML scores into existing model_scores hook — guardrails unchanged
  • Markov chain multi-touch attribution (already scaffolded)
  • Incrementality measurement via geo holdouts
  • LLM-powered conversational Q&A replacing regex patterns
Phase 3 6 – 12 months
Full Decisioning Loop
  • Auto-execute micro-adjustments (<5% shift) without human sign-off
  • Large shifts still require approval — trust ladder
  • Predictive budget forecasting ("approve this for 4 weeks → projected CAC $X")
  • Experiment orchestration: auto-launch holdouts, auto-call at p < 0.05
  • Creative fatigue wired to platform frequency cap APIs
Phase 4 12+ months
Scale & Intelligence
  • EMEA and APAC segment support
  • Budget scenario planning ("what if envelope +20%?")
  • Competitive intelligence feeds (SimilarWeb, SEMrush) wired to competition module
  • Shapley value attribution replacing heuristic
  • LTV model connected to real product cohort retention data
